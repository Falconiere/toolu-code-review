Create your Code Review GitHub App

One click pre-fills the name, the four least-privilege permissions, and disables the webhook. The App is created in your account — its private key never leaves you.

Must be globally unique on GitHub. Rename if taken (e.g. add your org). The App is created under this org, or your personal account if blank.
Permissions granted: Pull requests read & write · Issues read & write · Contents read · Metadata read. No webhook.

After GitHub creates the App

  1. Copy the App ID shown on the App's page.
  2. Click Generate a private key — it downloads a .pem.
  3. Install the App on the repo (or org) you want reviewed.
  4. Add two secrets to that repo/org (Settings → Secrets and variables → Actions): APP_ID and APP_PRIVATE_KEY — paste the whole PEM, or base64-encode it first (base64 -w0 key.pem) to keep it on one line; the action decodes either form.
  5. Pass them to the action with APP_ID: ${{ secrets.APP_ID }} and APP_PRIVATE_KEY: ${{ secrets.APP_PRIVATE_KEY }}.

Keep the .pem secret — anyone holding it can post as the App. Delete the download once it is stored as a secret.
